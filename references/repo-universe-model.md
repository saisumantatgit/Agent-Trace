# Repo Universe Model

The repo universe is a hybrid model:

- generated structure for what machines discover reliably
- curated overlays for what humans must define explicitly

The heart of the model is:

1. `search` helps generate the map
2. `manifest files` store the map
3. `agent query` uses the map before edits

For v1, generated structure should include:

- files
- symbols
- tests
- endpoints
- ADR docs

For v1, curated overlays should include:

- invariants
- ownership
- source-of-truth declarations
